Welcome to Galiphy!

Galiphy predicts new human genes that have a similar function to your genes of interest.

The only thing needed to run Galiphy for your research or clinical interest, is a simple list of human genes. This tool assumes that genes sharing a cellular function have a similar phenotype 'profile'.
By entering genes you are teaching the algorithm which phenotypes are typical for your genes of interest. Galiphy returns a list of scored genes. These scores depict how much each gene fits the phenotype 'profile'. High scoring genes that were not in your initial gene list are your candidate genes.

What does Galiphy do?

Super short explanation

Goal find new candidate genes associated with your cellular function of interest
Source database HPO (Human Phenotype Ontology) links phenotypes to {{ genes_in_HPO }} gene defects
Input list of genes that share a cellular function (do not enter candidate genes here)
Algorithm based on frequency of phenotypes among the input genes
Mid outcome all phenotypes in the HPO database are scored
Final outcome all {{ genes_in_HPO }} genes in the HPO database are scored
Conclusion genes with high scores probably have a similar cellular function as your input genes, and thus are your candidate genes

Further explanation:

  • Human Phenotype Ontology phenotype database (abbreviated as HPO)
  • Help page contains FAQ, tutorial and example section

Input: gene list

Gene list of interest

Accepted formats are gene names (symbol), Entrez gene IDs and Ensembl gene IDs, e.g. COX4I1P2, 652170 and ENSG00000228869, respectively.
The input genes must have a functional connection to each other. The more genes you provide, the better the algorithm predicts. For example, this tool was developed to find new mitochondrial proteins, i.e. genes not yet present in the Mitocarta list.
Note that Galiphy only uses genes listed in the phenotype database HPO as input. This means that on average only half the length of your gene list is run through the actual algorithm.

Output: 3 files

1. Phenotype scored

The phenotype scores are based on phenotype frequencies in the phenotype database HPO.
This file contains all phenotypes scored as well as their frequencies in and outside the input gene list.
Interesting for clinicians: find the most typical phenotypes of your query list.

2. Genes scored

The gene scoring is based on the previous file.
This file contains all {{ genes_in_HPO }} genes in HPO with their gene scores.
Most interesting file for researchers and clinicians: find your candidate genes, the high scoring genes outside the query.

3. How genes got their scores

This output file contains all {{ genes_in_HPO }} genes together with their HPO phenotypes, phenotype frequencies and phenotype scores.
This file is mostly for insight; see what Galiphy did between phenotypes scoring (file 1) and gene scoring (file 2). It shows how the algorithm scored the genes, i.e. adding all phenotype scores belonging to a gene.
