Reference & versions

Version 1.1

First update of Galiphy, resolving phenotype scoring shortcoming.
 [07-11-2018]
The phenotype scoring is adjusted for some cases, to see how normal scoring is done, look at the formula explanation.
Problem:
When phenotypes are not found among the query genes, the variable q is changed from 0 to 0.1. This is done in order to prevent mathematical error. However, 0.1 divided by any query gene list length still creates a bigger number than the value of nq divided by the non-query gene list length. I.e. in many cases, the phenotype score is higher than zero when a phenotype occurs zero times among the query genes. As this resulting positive phenotype score gives a gene an unwanted increase in gene score, a solution is needed.
Solution:
The highest unwanted phenotype score (PS) is the score of q=0 and nq=1. The scores are adjusted for all q = 0 cases by substracting the that PS. See the table below where both scoring in version 1.0 for q = 0 is shown and the new scoring in version 1.1.
(q;nq) PS1.0(n;nq) PS1.1(n;nq)
(0;1) 4.20 0.00
(0;2) 3.20 -1.00
(0;3) 2.00 -2.20
(0;x) PS1.0(0;x) PS1.0(0;x) - PS1.0(0;1)
