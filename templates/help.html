FAQs


Q1 How do I pronounce Galiphy?
The last syllable is pronounced the same as 'fly' but without the 'l'.

Q2 Is Galiphy worth exploring for me?
  • You are a clinical professional and interested in some candidate genes that may be associated with a disease.
  • You are a researcher and looking for candidate genes not yet discovered to be in your pathway of interest.
  • You are already working with genes that are underannotated and looking for more validation for your (clinical) lab experiments.

Q3 What kind of input is accepted?
Human genes in the format of:
  • Gene name (symbol): this format is not preferred as some genes have symbol synonyms.
  • Entrez gene ID: this is the advised format.
  • Ensemble gene ID: e.g. ENSG00000228869 and always without version number behind the dot (xxx.1 → xxx).
Separated by enters or spaces.

Q4 How do I get a gene list?
Good question. Larger gene lists get better predictions. Try to increase the size of your list by gathering genes that share a pathway, cellular function, cellular system, disease or co-expression. Maybe there is already a gene list there for your system of interest, like on WikiPathways, OMIM, WeGET, KEGG pathways, Reactome etc. Once you have your list of genes for a given biological process, you’re ready to see which phenotypes and other genes are associated using Galiphy.

Q5 Is my gene list too short?
Not all genes that you enter will run through the algorithm. Some genes will be dropped from the query because they are not listed in the phenotype database HPO. It is typical that around half the genes are dropped. It helps to start with at least 10 genes that are listed in HPO.
Galiphy tends to have better prediction power with the following factors:
  • Abundant phenotype annotation of the query genes
  • Similarity of the phenotypes among the query genes
  • Length of the query gene list
Take all of the above into consideration and always look at the output critically. As is typical with genomic tools, more genes is always better.

Q6 What is the easiest way to view the output files?
Open Excel, open the scoring files from here (File > Open) or right-click the file in your Download folder and click 'open in Excel'. Sorting the columns in Excel is a good way to explore the results of the analysis.

Q7 What can I do to validate the results?
Like in the Galiphy article, you can use several methods:

Statistical methods:
  • Statistical methods (with GSEA, ROC, Precision rank plot)
  • Literature
  • Functional correlation (with WeGET and STRING)
  • GSEA, gene set enrichment analysis.
Please see example results for a more in-depth explanation.

Tutorial

Step 1: Gene list

What Galiphy does:
Galiphy recognizes different types of gene entries (gene symbols, Entrez gene IDs and Ensembl gene IDs). Not all recognized genes are listed in the HPO database. Galiphy drops genes from your input gene list if they are not in HPO.
What you should do:
Provide a gene list using gene symbols, Entrez gene IDs or Ensembl gene IDs separated by enters or spaces. Click the submit button.


Step 2: Calculations

What Galiphy does:
Galiphy starts scoring. First, it generates a phenotype score for each phenotype in the database. Galiphy takes all phenotypes, determines how often each phenotype occurs in the query and how often it occurs in the rest of the HPO database. The phenotype score formula uses these frequencies to generate the phenotype score. Then, Galiphy adds all the scores of the phenotypes that belong to one gene which results in the gene score. More about scoring in step 4 and the algorithm section.
What you should do:
Wait 5 seconds.

Step 3: Results

What Galiphy does:
Galiphy writes your results in three files, the phenotype scores, the gene scores and the gene-to-phenotype file (the last file is mainly used for quality assessment).
What you should do:
Hit the download button to obtain all files; phenotype scores, the gene scores and the gene-to-phenotype file. It is important to take a look at the result page, in particular at column information of the files, input information and error alerts.

Step 4: Interpretation of the results

What Galiphy does:
He is done, he can rest now.
What you should do:
Import the scoring files in Excel (or right-click the file > 'open in Excel') or any other preferred program. And read about the files below.

The phenotype scoring file
The phenotype file shows you which phenotypes are most typical for your genes of interest. Besides the scores, you should also look at the frequency columns to understand how typical a phenotype is. The probability of finding any phenotype amongst the query list is dependent on the length of the query gene list. So HPO consists of {{ genes_in_HPO }} genes and if the query list (Q) contains 393 genes then the 'non-query list' (nQ) has {{ genes_in_HPO - 393 }} genes.

A high score means that the phenotype appears relatively often among your query genes. A good scoring phenotype could be linked to 29 of your query genes (q) and to none of the non-query genes (nq). This results in the Phenotype score of 11.28.

Sometimes you encounter frequencies of 0.1, these were zeroes that changed to 0.1 to prevent mathematical error in the formula.

The gene to phenotype file


This file allows you to see the phenotypes of each gene and how the addition of phenotype scores resulted in a gene score. If you click the image you can see the phenotype scores of the two query genes ATP5F1A and ATP5F1E.

The gene score is simply calculated by adding all the belonging phenotype scores.

You can observe how some phenotypes have little influence on the gene score, like autosomal recessive inheritance. Some have some influence, like nystagmus. Only a few have high influence, like high pitched cry and decreased activity of mitochondrial ATP synthase complex.


The gene scoring file
This file is the main result for most research questions.

Shows the scores of all {{ genes_in_HPO }} genes present in HPO. Galiphy sorted the genes on score. In the image, you can see the 17 highest scoring genes. The query column depicts whether the gene is (1) or is not (0) in your query gene list.

High scoring genes have phenotypes very similar to the phenotypes of your query genes. Therefore, the non-query genes ranked high in this file are your candidate genes.

Example


We made Galiphy with mitochondrial disorders (MD) as our research interest. Our goal was to find genes associated with the mitochondrion but were not yet listed in Mitocarta. Mitocarta2.0 consist of 1248 genes, which serves as the input list in this example. The proteins of these genes are known to be localized in the mitochondrion. After using Galiphy, clinical experts took these interesting candidate genes into consideration in diagnostics and experimental validation.


Input

Gene IDs over gene names
When you download Mitocarta2.0, you can easily select one column and copy-paste it into the input field of Galiphy on the home page. Copy-pasting the gene name column is allowed but will get you disappointed later as Galiphy tells you on the result page that a significant number of genes are not recognized. This is because some of the gene names are not up to date and are nowadays synonyms of genes.
Therefore, we recommend you to use the gene ID column.
Input evaluation
Galiphy can only run with the genes that are described in HPO. You can see which genes are found in HPO and which not on the result page. From the 1148 Mitocarta genes, 393 genes are listed in HPO. The other 765 genes are dropped from the algorithm.

Phenotype scores
All phenotypes in HPO are given a score. The phenotypes with the highest scores can be seen as the ones that are most typical for mitochondrial disorders (MD). You should look critically at this file because it could give you interesting information:

Using clinical knowledge to edit the query gene list
If some phenotypes are not common for the disease according to your clinical knowledge, consider eliminating the genes that are disrupting the quality of the phenotype profile from your query list in order to get more reliable results.
In our study, SDH gene defects caused two specific types of cancer. All six SDH genes were therefore removed from the query list. Galiphy was run again which resulted in better phenotype scoring.

Look for interesting numbers
When a phenotype is clinically relevant to MD and only a few non-query genes assigned to it, pay attention to the non-query genes as they could be interesting genes. Find more information about these genes in the next file, 'genes to phenotypes'.

'Useless' phenotypes
Some phenotypes only occur once in the HPO database. These phenotypes are not valuable for the scoring of the genes because they do not give information about how typical a phenotype is.

Make plots
Get an insight on how the phenotype scores are distributed and which scores are exceptionally high or low by making a distribution plot of the scores.



Genes to phenotypes
This file lists all genes in HPO together with their phenotypes and phenotype scores. It tells you how each gene got its gene score as the score results from adding all phenotype scores. Most likely, you will look at this file after you have curated the gene score or the phenotype file. You could look for the phenotypes with which one specific gene of interest is associated with.

Phenotype to gene association
If you sort on phenotypes you can see which genes are assigned to that specific phenotype of interest. This way, you created the reversed file, the phenotype to gene file.


Gene scores
Most important file for our research question. The genes that have a high gene score and are not in Mitocarta are our candidate genes. These genes got their high scores because the phenotypes they are associated with are similar to the phenotypes of the input genes.

Function of single candidate genes
Use databases like GeneCards to get an overview of function, localization and diseases the gene is associated with. The example file shows you that the highest scoring non-query gene is TRNS1, a mitochondrially encoded tRNA Serine gene.

Analysis of multiple candidate genes
Look for correlation between the candidate genes. You could take the best 200 scoring non-query genes and put them into online tools like WeGET co-expression or with String's enrichment analysis for KEGG pathways and GO biological processes. In the Galiphy article you can find the results of these tools in Appendix V and VI.

Edit your gene list again
To make a more specific 'phenotype fingerprint' the query gene list was adjusted again (the first time is described in the phenotype file explanation above). This time, genes in the oxidative phosphorylation (OXPHOS) pathway were put in the query gene list in order to optimize prediction quality.

Make plots
A distribution plot like Figure 4A and B shows you how your query genes score in comparison to the non-query genes. An overview of how the genes are ranked can be seen in Figure 5B .



Statistical plots for validation of the prediction
If you would like to use statistical methods to measure the quality of the Galiphy results, you can use the following analyses:

  • GSEA, gene set enrichment analysis.
  • ROC, receiver operating characteristic curve. The area under the curve (AUC) can be used as a prediction evaluation.
  • Precision rank plot: precision (positive prediction) value vs. rank.
These can also be found in Figure 5ACD of the Galiphy article.

Algorithm



Galiphy’s algorithm is based on the enrichment of phenotypes in the input list of genes.
Phenotypes are scored using the following principles:
  1. Phenotypes that are more often associated with the query genes should score higher than phenotypes that are more often associated with the non-query genes.
  2. If, for instance, HPO annotates 75 genes of the query, the remaining {{ genes_in_HPO - 75 }} genes are automatically defined as non- query genes.
  3. Prior is taken into account, meaning that the relative chances of finding phenotypes in each query list are incorporated.

After the client provides an input gene list, all HPO phenotypes are gathered of those genes and of the genes that are not input genes. Then, each phenotype is given a score:
